01 About

These days I work in fenestration: windows, glazing, and the codes and simulations that decide how well they perform. That means NFRC / CSA A440.2 compliance, thermal simulation, building out ISO/IEC 17065 QMS and EPD/LCA programs, and writing automation that turns multi-week simulation work into a couple of days.

I got here by a fairly winding road (kitchens, golf course irrigation, a flower farm, drywall, even a stint reffing minor hockey) before circling back to school for engineering and landing in building science. I still like the hands-on stuff; these days that's Python scripts and thermal models instead of pizza ovens and pipe fittings.

I'm active in NFRC's standards process as an ANS Standards Committee member and Task Group presenter, where I've presented independent technical research on two-box spacer modeling methodology and validation, and I hold both NFRC LEAFF and Phius Blue Path simulator certifications (see Skills below).

03 Projects & Research

Two-Box Spacer Modeling Validation

Independent technical report evaluating two-box spacer calibration strategies against detailed spacer models using LBNL THERM and NFRC 100/200/500 methodology: 900 full-frame simulations across 3 spacer families and 25 widths (8-20 mm), holding U-factor deviations within ~0.008 W/m²·K. Presented to the NFRC Simulators Task Group.

  • LBNL THERM
  • NFRC 100/200/500
  • Research

NFRC-101-Materials

Automated pipeline that monitors, parses, and validates NFRC 101 material property tables (Appendices A, B, C) from the published PDF, syncing them into structured, versioned JSON via a scheduled GitHub Actions workflow.

  • Python
  • GitHub Actions
  • Data Pipeline

Thm-Reader

Rust library for reading and writing THERM 7.x binary .thm files, with typed access to header, polygon, and IGU/glazing data plus the material, boundary-condition, and gas libraries, with round-trip byte-accurate serialization and JSON export.

  • Rust
  • Binary Parsing
  • Open Source

Simulation Database Platform

Web-based simulation database (Next.js/Supabase) cataloguing all historical project simulations, with automatic integration into calculation spreadsheets and reporting workflows.
